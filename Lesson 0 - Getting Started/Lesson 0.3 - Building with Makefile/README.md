# Урок 0.3 — Организация сборки SDL2 проекта для Linux и PlayStation 2

В этом уроке мы разделим систему сборки на несколько Makefile-файлов и создадим удобную структуру для поддержки нескольких платформ.

На данный момент проект поддерживает:

* Linux
* PlayStation 2 Homebrew

В будущем по аналогии можно добавить:

* Windows
* PSP
* PS Vita
* Android

## Структура проекта

```
sdl2-app-skeleton/
├── bin/
├── src/
│   └── main.c
├── Makefile
├── Makefile.linux
└── Makefile.ps2
```

## Главный Makefile

Файл Makefile является точкой входа для сборки проекта.

Он не выполняет компиляцию самостоятельно, а перенаправляет управление в платформозависимые Makefile.

```makefile

.PHONY: all linux ps2 clean

all: linux ps2

linux:
	$(MAKE) -f Makefile.linux

ps2:
	$(MAKE) -f Makefile.ps2

clean:
	$(MAKE) -f Makefile.linux clean
	$(MAKE) -f Makefile.ps2 clean

```

### Сборка всех платформ

```bash
make
```

Будут собраны:

```
bin/game_linux
bin/game_ps2.elf
```

### Сборка только Linux версии
```bash
make linux
```

#### Сборка только PS2 версии
```bash
make ps2
```

#### Очистка проекта
```bash
make clean
```

Команда вызывает очистку для всех платформ.


## Makefile.linux

Данный файл отвечает за сборку Linux-версии приложения.

```makefile
CC = gcc

CFLAGS = -O2 -Wall -DPLATFORM_LINUX `sdl2-config --cflags`
LDFLAGS = `sdl2-config --libs`

SRC = src/main.c
OUT = bin/game_linux
```

---

### Используемые параметры

```Makefile
CC = gcc
```

Переменная CC хранит имя компилятора.

В данном случае используется:

```
gcc
```

Это компилятор языка C из набора GNU Compiler Collection (GCC).

Потом вместо слова gcc можно писать
```
$(CC)
```
Например
```
$(CC) main.c
```
равносильно
```
gcc main.c
```
Если позже понадобится использовать другой компилятор, достаточно изменить одну строку:
```
CC = clang
```
или
```
CC = mips64r5900el-ps2-elf-gcc
```
для PlayStation 2.


---

### CFLAGS

```makefile
CFLAGS = -O2 -Wall -DPLATFORM_LINUX `sdl2-config --cflags`
```
Эта переменная содержит флаги компиляции.

Когда выполняется команда
```
gcc $(CFLAGS)
```
она превращается примерно в
```
gcc -O2 -Wall -DPLATFORM_LINUX -I/usr/include/SDL2 -D_REENTRANT
```
Разберем каждый параметр.

```
-O2
```
Включает оптимизацию кода.

Есть несколько уровней:

**Без оптимизации**
```
-O0
```
По умолчанию.

Компилируется быстро.

Удобно для отладки.

**Небольшая оптимизация**
```
-O1
```

Компилятор делает простые оптимизации.

**Хорошая оптимизация**
```
-O2
```
Наиболее популярный вариант.

Даёт хороший прирост скорости без слишком долгой компиляции.

*Именно его используют чаще всего.*

**Максимальная оптимизация**
```
-O3
```
Компилятор делает ещё больше оптимизаций.

Иногда программа становится быстрее.

Иногда наоборот.

`Оптимизация размера`

```
-Os
```

Компилятор старается уменьшить размер программы.

Полезно для встроенных устройств.

---

`-Wall`

Включает почти все полезные предупреждения.

Например:

неиспользуемые переменные

```
int x = 5;
```

Компилятор скажет

```bash
warning: unused variable 'x'
```

Отсутствие return

```cpp
int foo()
{
}
```

Получим предупреждение:

- Подозрительные преобразования типов.

- Ошибки в условиях.

Практически всегда рекомендуется использовать

```
-Wall
```

---

`-DPLATFORM_LINUX`

```
-D
```

означает

   определить макрос.

То есть компилятор ведёт себя так, словно в начале каждого файла написано

```cpp
#define PLATFORM_LINUX
```

После этого можно писать

```cpp
#ifdef PLATFORM_LINUX

printf("Linux\n");

#endif
```

или

```cpp
#ifndef PLATFORM_LINUX
```

Очень удобно для кроссплатформенного кода.

Например
```cpp
#ifdef PLATFORM_PS2

...

#else

...

#endif

```

Можно передавать значения.

Например

```bash
-DWIDTH=800
```

получится
```cpp
#define WIDTH 800
```

---

`sdl2-config --cflags`

Это уже не флаг GCC.

Это выполнение команды оболочки.

Команда

```bash
sdl2-config --cflags
```

возвращает что-то вроде

```bash
-I/usr/include/SDL2 -D_REENTRANT
```

Именно это автоматически подставляется в команду компиляции.

---

`-I`

Например
```
-I/usr/include/SDL2
```
означает

искать заголовочные файлы ещё и здесь.

Именно поэтому работает
```cpp
#include <SDL2/SDL.h>
```
Без `-I`

компилятор мог бы написать

```
SDL.h not found
```

`-D_REENTRANT`

SDL автоматически добавляет ещё один макрос.

Он нужен для корректной работы потоков.

Обычно о нём думать не нужно.


---

### LDFLAGS

```makefile
LDFLAGS = `sdl2-config --libs`
```

Это флаги **линковщика**.

После компиляции объектных файлов начинается этап линковки.

Компоновщик должен найти реализации функций.

Например

```cpp
SDL_Init();
```

объявлена в

```cpp
SDL.h
```

но сама функция находится внутри библиотеки SDL2.

Поэтому нужно написать

```bash
-lSDL2
```

Именно это возвращает

```bash
sdl2-config --libs
```

Например

```
-L/usr/lib/x86_64-linux-gnu -lSDL2
```
---

`-L`

Например

```bash
-L/usr/lib
```

означает

искать библиотеки ещё и здесь.

`-l`

Например

```bash
-lSDL2
```

говорит линковщику

подключить библиотеку SDL2.

Он автоматически найдёт

```
libSDL2.so
```

или

```
libSDL2.a
```
---

### SRC

```Makefile
SRC = src/main.c
```

Список исходных файлов.

Пока только один.

Можно добавить несколько.

```Makefile
SRC = \
  src/main.c \
  src/input.c \
  src/player.c
```
---

`OUT`

```
OUT = bin/game_linux
```

Имя выходного файла.

После компиляции получится

```
bin/game_linux
```
---

### Цель all
```
all:
```
Это основная цель.

Когда написать

```bash
make
```
будет выполнена именно она.

Команда

```
$(CC) $(CFLAGS) $(SRC) -o $(OUT) $(LDFLAGS)
```

После подстановки переменных превращается примерно в

```bash
gcc \
-O2 \
-Wall \
-DPLATFORM_LINUX \
-I/usr/include/SDL2 \
-D_REENTRANT \
src/main.c \
-o bin/game_linux \
-L/usr/lib \
-lSDL2
```

`-o`

```bash
-o bin/game_linux
```

указывает имя выходного файла.

Без него GCC создаст файл

```
a.out
```

Поэтому почти всегда используют

```
-o имя
```

### Цель clean

```
clean:
```

Это соглашение.

Она удаляет результаты сборки.

Команда

```
rm -f $(OUT)
```

становится

```
rm -f bin/game_linux
```

`rm`

Удалить файл.

---

`-f`

Флаг `-f` означает **force** ("принудительно").

Он делает две вещи:

* не спрашивает подтверждения при удалении;
* не выводит ошибку, если файла не существует.

Поэтому команду `make clean` можно выполнять даже тогда, когда проект ещё не собирался.

### Итог: что происходит при make

1. Подставляется CC → gcc.

2. Добавляются флаги компиляции (CFLAGS):

  	* -O2 — оптимизация;
  	* -Wall — предупреждения;
  	* -DPLATFORM_LINUX — определение макроса;
  	* -I... и -D_REENTRANT — параметры, возвращаемые sdl2-config --cflags.

3. Компилируется src/main.c.
4. Флаг -o задаёт имя выходного файла bin/game_linux.
5. На этапе линковки используются параметры из LDFLAGS, например -L... и -lSDL2, чтобы подключить библиотеку SDL2.
6. В результате создаётся исполняемый файл bin/game_linux.

---

### Результат сборки

После выполнения:

```bash
make -f Makefile.linux
```

будет создан файл:

`bin/game_linux`


## Makefile.ps2

Данный файл отвечает за сборку PlayStation 2 Homebrew приложения.

### PS2DEV и PS2SDK

```makefile
PS2DEV ?= /usr/local/ps2dev 
PS2SDK ?= $(PS2DEV)/ps2sdk
```

Определяют пути к установленному тулчейну.

`?=` означает:

присвоить значение только если переменная ещё не определена.

То есть если пользователь выполнит

```bash
make PS2DEV=/home/vitya/ps2dev
```

это заменит PS2DEV


### export PATH
```Makefile
export PATH := $(PATH):$(PS2DEV)/ee/bin
```
Здесь сразу несколько новых вещей.

`export`

Делает переменную окружения доступной дочерним процессам.

Например, когда Make вызывает компилятор.


`PATH`

Это системная переменная окружения.

Она содержит список каталогов, где система ищет программы.

Например

```
/usr/bin
/bin
/usr/local/bin
```
Когда вы пишете

```bash
gcc
```

Linux ищет файл

```
/usr/bin/gcc
```

Здесь к PATH добавляется

```
/usr/local/ps2dev/ee/bin
```

Именно там лежат

```
mips64r5900el-ps2-elf-gcc
mips64r5900el-ps2-elf-ld
mips64r5900el-ps2-elf-ar
...
```

Теперь Make может запускать их без полного пути.

---

`:=`

В Make есть несколько способов присваивания.

Здесь используется

```
:=
```

Это немедленное присваивание.

То есть
```
$(PATH)
```

раскрывается сразу.

### EE_BIN

```makefile
EE_BIN = bin/game_ps2.elf
```

Имя итогового ELF-файла.

После сборки получится
```
bin/game_ps2.elf
```

#### Что такое ELF?

ELF — это стандартный формат исполняемых файлов в Linux и PS2.

Он содержит:

* код программы;
* данные;
* таблицу символов;
* информацию для загрузчика.

Позже этот ELF можно запускать через:

* uLaunchELF;
* ps2link;
* Open PS2 Loader (в некоторых сценариях).

### EE_CFLAGS

```
EE_CFLAGS += -DPLATFORM_PS2
```
+= означает

добавить к существующему содержимому.

То есть если PS2SDK уже добавил

```
-G0 -O2
```

после этой строки получится

```
-G0 -O2 -DPLATFORM_PS2
```

Макрос

```
PLATFORM_PS2
```

позволяет писать

```
#ifdef PLATFORM_PS2
```

### EE_SRCS
```Makefile
EE_SRCS := $(wildcard src/*.c)
```
Здесь используется функция

```
wildcard
```

Она ищет все файлы, подходящие под шаблон.

Например

```
src/main.c
src/input.c
src/text.c
```

Получится
```
EE_SRCS =
src/main.c
src/input.c
src/text.c
```
Очень удобно.

Не нужно каждый раз вручную добавлять новые файлы.

### EE_OBJS

```makefile
EE_OBJS := $(EE_SRCS:.c=.o)
```

Это преобразование списка.

Было

```
src/main.c
src/input.c
```
Стало
```
src/main.o
src/input.o
```

Компилятор сначала создаёт объектные файлы.

Потом линковщик объединяет их.

### EE_INCS

```Makefile
EE_INCS += -I$(PS2SDK)/ports/include
```
Добавляет путь поиска заголовков.

После подстановки
```
-I/usr/local/ps2dev/ps2sdk/ports/include
```
Именно там находятся
```
SDL2
SDL2_ttf
png
freetype
...
```

Следующая строка
```Makefile
EE_INCS += -I$(PS2DEV)/gsKit/include
```

Добавляет заголовки gsKit.


### EE_LDFLAGS

```Makefile
EE_LDFLAGS += -L$(PS2SDK)/ports/lib
```

Добавляет каталог библиотек.

После раскрытия
```
-L/usr/local/ps2dev/ps2sdk/ports/lib
```

Следующая строка

```Makefile
EE_LDFLAGS += -L$(PS2DEV)/gsKit/lib
```

Добавляет библиотеки gsKit.

### EE_LIBS

Это список библиотек.

`-lSDL2_ttf`

Поддержка TrueType-шрифтов. Позволяет использовать `TTF_OpenFont()`

`-lfreetype`

Библиотека, которая умеет читать TTF.

SDL_ttf использует её внутри.


`-lpng`

Загрузка PNG.

`-lz` (zlib)
Распаковка данных.

PNG использует zlib.

`-lSDL2`

Основная библиотека SDL.

Здесь находятся
```
SDL_Init()
SDL_CreateWindow()
SDL_RenderPresent()
...
и многое другое.
```

`-lps2_drivers`

Набор драйверов PS2.

`-lgskit`

Работа с графическим процессором PS2.

SDL2 для PS2 использует gsKit как графическую основу.

`-ldmakit`

Работа с DMA.

DMA позволяет быстро копировать данные между памятью и графическим процессором.


`-laudsrv`

Работа со звуком.

SDL Audio использует именно его.

`-lpadx`

Работа с геймпадами.

`-lmtap`

Поддержка Multitap.

Позволяет подключить больше двух контроллеров.

`-lmouse`

Поддержка USB-мыши.


`-lfileXio`

Расширенная файловая система PS2.

Используется для доступа к USB, HDD, сети и другим устройствам.

`-lpatches`

Различные патчи и исправления среды выполнения PS2.

`-ldebug`

Отладочные функции.

Например вывод сообщений через ps2link.


### include

```makefile
include $(PS2SDK)/samples/Makefile.pref
```
Эта строка подключает ещё один Makefile.

Можно представить это так, будто его содержимое вставляется сюда.

В нём находятся:

* выбор компилятора;
* стандартные флаги;
* пути к SDK;
* различные настройки сборки.

Следующий

```makefile
include $(PS2SDK)/samples/Makefile.eeglobal
```

Это основной Makefile PS2SDK.

Именно он:

* компилирует все .c;
* создаёт .o;
* запускает линковщик;
* собирает ELF;
* знает, как использовать переменные EE_SRCS, EE_OBJS, EE_LIBS, EE_INCS, EE_CFLAGS, EE_LDFLAGS и другие.

Именно поэтому в нашем Makefile нет цели all — её предоставляет `Makefile.eeglobal`.


### clean

```makefile
clean:
	rm -f $(EE_BIN) $(EE_OBJS)
```

Удаляет:

```
bin/game_ps2.elf
```

и

src/main.o
src/input.o
src/text.o
...

Флаг -f позволяет выполнять очистку без ошибок, даже если файлов уже нет.

### Как проходит сборка

Когда вы выполняете:

```
make 
```

происходит примерно следующее:

1. Задаются пути к `PS2DEV` и `PS2SDK`.
2. В `PATH` добавляется каталог с инструментами PS2 (`ee/bin`).
3. Находятся все исходники `src/*.c` (`wildcard`).
4. Из списка исходников автоматически формируется список объектных файлов (`.o`).
5. Добавляются пути к заголовочным файлам (`EE_INCS`) и библиотекам (`EE_LDFLAGS`).
6. Указывается набор библиотек, необходимых для SDL2, графики, звука, ввода и файловой системы (`EE_LIBS`).
7. Подключается стандартная система сборки PS2SDK (`Makefile.pref` и `Makefile.eeglobal`).
8. `Makefile.eeglobal` компилирует каждый `.c` в `.o`, затем линкует их в итоговый файл `bin/game_ps2.elf`.


### Результат сборки

После выполнения:

`make -f Makefile.ps2`

будет создан файл:

`bin/game_ps2.elf`


## Добавление новых платформ

Текущая архитектура позволяет легко расширять проект.

Например, для Windows можно добавить:

`Makefile.windows`

и расширить главный Makefile:

`all: linux windows ps2`

После чего станет доступна команда:

`make windows`

---

## Итог

После выполнения урока вы получите универсальную систему сборки SDL2-проекта с разделением по платформам.

Такой подход позволяет поддерживать несколько платформ одновременно, не усложняя основной Makefile и сохраняя код проекта максимально чистым и удобным для сопровождения.
