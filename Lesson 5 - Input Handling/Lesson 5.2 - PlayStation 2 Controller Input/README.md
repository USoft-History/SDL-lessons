# Lesson 4.2 - PlayStation 2 Controller Input

В этом уроке мы научимся работать с контроллером PlayStation 2 через библиотеку **libpad** из состава PS2SDK.

На данный момент SDL2 для PlayStation 2 не предоставляет полноценную поддержку игровых контроллеров через стандартные механизмы SDL, поэтому для чтения кнопок DualShock используется API PS2SDK.

После изучения урока вы сможете:

* Инициализировать систему контроллеров PS2.
* Подключать DualShock к приложению.
* Считывать состояние кнопок.
* Реагировать на нажатия крестовины и основных кнопок.
* Использовать контроллер для управления игровыми объектами.

## Подключение заголовочных файлов

Для работы с контроллером потребуются следующие заголовочные файлы:

```c
#include <tamtypes.h>
#include <kernel.h>
#include <sifrpc.h>
#include <loadfile.h>
#include <libpad.h>
```

## Буфер контроллера

Создадим буфер и структуру для хранения состояния кнопок:

```c
static u8 padBuf[256] __attribute__((aligned(64)));
static struct padButtonStatus buttons;
```

Буфер должен быть выровнен по границе 64 байта.

---

## Инициализация системы контроллеров

Перед использованием контроллера необходимо загрузить системные модули PS2:

```c
SifInitRpc(0);

SifLoadModule("rom0:SIO2MAN", 0, NULL);
SifLoadModule("rom0:PADMAN", 0, NULL);

padInit(0);
```

Здесь:

1. SIO2MAN предоставляет низкоуровневый доступ к интерфейсу контроллеров.
2. PADMAN реализует высокоуровневый драйвер геймпада.
3. padInit() инициализирует библиотеку libpad.


`void SifInitRpc(int mode)` — это функция из PS2SDK, которая инициализирует подсистему RPC (Remote Procedure Call) между процессорами PlayStation 2.

---

`int SifLoadModule(const char *path, int arg_len, const char *args)` - загружает модуль на процессор IOP (Input Output Processor).

Многие подсистемы PlayStation 2 реализованы в виде отдельных модулей: работа с контроллерами, картами памяти, USB, сетью, файловой системой и т.д.
Перед использованием такой подсистемы необходимо загрузить соответствующий модуль.

Параметры

**path**

Путь к модулю, который необходимо загрузить.

**arg_len**

Размер строки аргументов в байтах.

**args**

Строка аргументов для модуля.

**Возвращаемое значение**

Функция возвращает:

    >= 0 — модуль успешно загружен.

    < 0 — ошибка загрузки.

---

`int padInit(int mode)` - Функция padInit() инициализирует библиотеку libpad, которая используется для работы с контроллерами PlayStation 2.

После загрузки модулей SIO2MAN и PADMAN необходимо вызвать padInit(), чтобы подготовить библиотеку к работе.

Параметр 0 означает стандартную инициализацию библиотеки.

**Функция возвращает:**

`1`

если инициализация выполнена успешно.

В случае ошибки возвращается отрицательное значение.

---

## Открытие порта контроллера

После инициализации открываем первый порт контроллера:

```c
if (!padPortOpen(0, 0, padBuf))
{
    printf("padPortOpen failed\n");
    return 1;
}
```

Параметры:

| Значение | Описание          |
| -------- | ----------------- |
| 0        | Первый порт       |
| 0        | Первый слот       |
| padBuf   | Буфер контроллера |

---

## Проверка состояния контроллера

Перед чтением кнопок необходимо убедиться, что контроллер готов к работе:

```c
int state = padGetState(0, 0);

if (state != PAD_STATE_STABLE &&
    state != PAD_STATE_FINDCTP1)
{
    return;
}
```

---

## Чтение кнопок

Состояние кнопок считывается функцией:

```c
padRead(0, 0, &buttons);
```

Полный пример:

```c
if (padRead(0, 0, &buttons) > 0)
{
    u16 btns = ~buttons.btns;

    if (btns & PAD_UP)
        SDL_Log("UP");

    if (btns & PAD_DOWN)
        SDL_Log("DOWN");

    if (btns & PAD_LEFT)
        SDL_Log("LEFT");

    if (btns & PAD_RIGHT)
        SDL_Log("RIGHT");

    if (btns & PAD_CROSS)
        SDL_Log("CROSS");

    if (btns & PAD_CIRCLE)
        SDL_Log("CIRCLE");
}
```

## Особенности SDL2 на PlayStation 2

На момент записи урока рекомендуется использовать следующую инициализацию SDL:

```c
SDL_Init(SDL_INIT_VIDEO);
```

Использование:

```c
SDL_Init(SDL_INIT_VIDEO |
         SDL_INIT_AUDIO |
         SDL_INIT_JOYSTICK);
```

может приводить к зависанию приложения на PlayStation 2.

Поэтому работу с контроллером рекомендуется выполнять через `libpad`.
