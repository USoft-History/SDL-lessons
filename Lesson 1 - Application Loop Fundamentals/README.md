В этом уроке мы создадим первое полноценное SDL2-приложение и разберём основные элементы, которые используются практически в каждой игре.

## Структура SDL2-приложения

Практически любая SDL2-программа состоит из нескольких этапов:

1. Инициализация SDL.
2. Создание окна.
3. Создание рендера.
4. Запуск главного цикла.
5. Обработка событий.
6. Отрисовка кадра.
7. Завершение работы приложения.

## Разбор кода

**SDL_Init**

SDL_Init(SDL_INIT_VIDEO);

Инициализирует видеоподсистему SDL.

Без вызова этой функции работа с окном и рендерингом невозможна.

---

**SDL_CreateWindow**

```SDL_CreateWindow(...)```

Создаёт окно приложения.

Основные параметры:
* Заголовок окна
* Позиция на экране
* Ширина
* Высота
* Дополнительные флаги

---

**SDL_CreateRenderer**

```SDL_CreateRenderer(...)```

Создаёт объект рендера.

Через рендер выполняется вся последующая отрисовка графики.

---

**Главный цикл**
```cpp
while (running)
{
}
```

Главный цикл выполняется до тех пор, пока приложение не будет закрыто.

Именно внутри него происходит:

* обработка ввода;
* обновление игровой логики;
* отрисовка кадра.

Главный цикл является основой любой игры.

---

**SDL_PollEvent**
```SDL_PollEvent(&event);```

Получает события от операционной системы.

Например:

* закрытие окна;
* нажатие клавиш;
* движение мыши;
* работа геймпада.

В нашем примере мы обрабатываем событие закрытия окна:
```cpp
if (event.type == SDL_QUIT)
{
    running = false;
}
```

---


**Очистка экрана**

Перед отрисовкой нового кадра необходимо очистить экран.

```cpp
SDL_SetRenderDrawColor(renderer, 0, 0, 0, 255);
SDL_RenderClear(renderer);
```

В данном случае экран очищается чёрным цветом.

---

**SDL_RenderPresent**

```SDL_RenderPresent(renderer);```

Показывает пользователю готовый кадр.

Все команды рисования сначала выполняются во внутреннем буфере, а затем одним вызовом отображаются на экране.

Этот механизм называется двойной буферизацией.

---

**Завершение приложения**

После выхода из главного цикла необходимо освободить все созданные ресурсы.

```cpp
SDL_DestroyRenderer(renderer);
SDL_DestroyWindow(window);
SDL_Quit();
```

Правильное освобождение ресурсов помогает избежать утечек памяти и других проблем.
